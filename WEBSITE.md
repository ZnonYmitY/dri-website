# 网站维护说明

## 公开发布方案

本项目可以通过 GitHub 解决公开访问问题。推荐使用 GitHub Pages：把网站代码推送到 GitHub 仓库后，在仓库设置中开启 Pages，即可生成一个公开 URL，其他人可以直接访问。

当前线上信息：
- GitHub 仓库：https://github.com/ZnonYmitY/dri-website
- 公开访问链接：https://znonymity.github.io/dri-website/
- 当前发布方式：GitHub Pages，从 `main` 分支根目录发布。

适用场景：
- 静态网站：HTML、CSS、JavaScript、React/Vue/Vite 构建后的前端页面。
- 展示型网站：个人主页、项目介绍页、活动页、作品集、文档站。
- 不需要后端数据库或私有服务的轻量应用。

不适用或需要额外方案的场景：
- 需要长期运行后端服务。
- 需要数据库、登录态、服务端接口或复杂权限控制。
- 需要隐藏源码或部署私有业务逻辑。

如果后续网站需要后端能力，可以保留 GitHub 托管代码，同时把线上部署迁移到 Vercel、Netlify、Cloudflare Pages 或自有服务器。

## GitHub Pages 操作流程

1. 在 GitHub 创建一个新仓库，例如 `dri-website`。
2. 在本地项目中绑定远程仓库：

```bash
git remote add origin git@github.com:<your-name>/dri-website.git
```

3. 提交并推送代码：

```bash
git add .
git commit -m "Initial website setup"
git push -u origin main
```

4. 打开 GitHub 仓库页面，进入 `Settings` -> `Pages`。
5. 在 `Build and deployment` 中选择发布方式：
   - 如果是纯静态 HTML：选择 `Deploy from a branch`，分支选 `main`，目录选 `/(root)`。
   - 如果是 Vite/React 等构建型项目：选择 `GitHub Actions`，配置自动构建并发布 `dist` 目录。
6. 保存后等待 GitHub Pages 构建完成。
7. 获得公开访问链接，通常格式为：

```text
https://<your-name>.github.io/dri-website/
```

## 网站内容概要

- 网站名称：AI DRI 指南针。
- 目标用户：希望从“会用 AI”进一步成为 AI 时代 DRI 的业务负责人、项目负责人和 AI 实践者。
- 核心用途：用导览、自测和定义澄清，帮助用户理解 AI DRI 的能力结构、责任边界和价值闭环。
- 首页传达的第一信息：从会用 AI，到成为 AI 时代的 DRI。
- 主要页面：当前为单页网站；顶部 tab 已按未来子页面入口预留。
- 重要内容模块：首页首屏、右侧能力卡片、自测题、下方滑出结果卡片、DRI 是 / 不是、问题定义浮窗、指南针说明。

## 功能特点

当前状态：已具备 Claude-like warm editorial 首页和浏览器内编辑能力。

- 核心功能：公开访问、文字可编辑、图片上传替换、一键保存、导出新 HTML、自测题、结果卡片、定义浮窗。
- 辅助功能：顶部导航子页面占位、页面内 CTA 锚点跳转、响应式布局、右上角编辑模式工具栏。
- 视觉风格：参考 Claude 的温暖画布、克制卡片、serif 大标题、深色产品展示卡片；主色改为新桥青 `#1661ab`，底色为宣纸白 `#f9f1db`，清水蓝 `#93d5dc` 和玉林紫 `#a7a8bd` 作为辅助色，字节蓝 `#165dff` 仅保留为少量进度高亮。
- 数据来源：当前内容来自项目内静态 HTML 和本维护文档。
- 是否需要搜索、筛选、排序：当前暂不需要。
- 是否需要表单、下载、分享：支持导出 HTML 下载；当前暂不需要表单。
- 是否需要适配移动端：需要，首版已做基础响应式适配。

## 交互细节

当前状态：已实现首版重点交互。

- 首屏交互：点击“开始导览”直接跳到“DRI 到底是什么”；点击“先做自测”跳到自测模块。
- 导航方式：顶部固定导航；当前 tab 为未来子页面占位，页面内 CTA 通过锚点跳转。
- 编辑模式：右上角“编辑模式”开启后，页面文字进入可编辑状态，图片显示上传入口；保存会写入浏览器 localStorage，导出会下载新的 HTML 文件。
- 自测交互：7 个能力维度各 1 道题，每题 3 个选项，分别代表入门、不错、精通；全部选择后在自测区下方插入滑出评级结果卡片，不再使用弹窗关闭交互。
- DRI 定义交互：鼠标悬停右侧“问题定义”等标签会浮现说明，移开后消失。
- FAQ 交互：“为什么需要这套指南针”采用右侧加号，点击后加号旋转并展开说明内容。
- 按钮和链接行为：非编辑模式下锚点正常跳转；编辑模式下页面内链接禁止跳转，避免编辑文字时误触。
- 表单校验：首版没有传统表单；图片上传限制为 image 类型。
- 加载、空状态和错误状态：首版为静态内容，暂不涉及异步加载状态。
- 移动端交互：导航横向滚动，内容模块改为单列，编辑按钮保持右上角。
- 键盘或无障碍支持：支持 Escape 关闭定义浮窗，页面使用语义化区域、导航标签和可聚焦按钮。
